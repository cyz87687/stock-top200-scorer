# A股TOP200评分报告生成器

## 用途

生成基于成交额TOP200的A股个股多维度评分HTML报告，包含消息面、技术面、基本面、题材热度四维评分体系。

## 触发条件

当用户要求以下任一操作时调用：
- "跑今日数据" / "跑一下数据" / "更新数据"
- "生成报告" / "生成HTML" / "更新report"
- "评分" / "打分" / "股票评分"

## 执行步骤

### Step 0: 确认工作目录

```bash
cd /Users/yzreal/WorkBuddy/2026-06-11-20-38-35
```

### Step 1: 获取TOP200行情数据

```bash
python fetch_top200_sina.py
```

- 数据源：新浪财经全市场行情接口
- 输出：`top200_all_a.json`（含name/code/price/pct_chg/turnover）
- 如已存在当日数据可跳过此步

### Step 2: 多维度评分

```bash
python step2_stockscorer_v2.py
```

- 输入：`top200_all_a.json`（默认）
- 输出：`top200_scored_YYYY-MM-DD.json`
- 耗时约10-15分钟（190只股票，串行获取K线+周线数据）

#### 评分流程：
1. **K线获取**：腾讯前复权API（主）→东方财富（备）→新浪（兜底），串行+间隔控制
2. **周线获取**：腾讯前复权周线API
3. **板块行情**：新浪行业板块接口获取板块涨跌幅
4. **iFinD新闻**：重点标的新闻关键词分析
5. **四维评分**：
   - 题材热度(30%)：基础分+板块涨跌幅+涨停验证+资金流入+生命周期
   - 基本面(30%)：FwdPE+增速+PEG+ROE+毛利率+估值阶梯+行业前景
   - 消息面(20%)：已知催化剂+iFinD新闻+股价确认，含来源/时间/情感标签
   - 技术面(20%)：双线多头(日线+周线)+BOLL+价格分位+量价配合+支撑压力位
6. **评级**：S(≥16) / A(≥13) / B(≥10) / C(≥7) / D(<7)

### Step 3: 生成HTML报告

```bash
python gen_report_top200.py
```

- 输入：`top200_scored_YYYY-MM-DD.json`（默认当日）
- 输出：`top200_report_YYYY-MM-DD.html`
- 报告包含：评级分布统计、板块均分图表、个股详情卡片、组合推荐

## 四维评分体系详解

### 1. 题材热度 (0-5分, 权重30%)
- 基础分：THEME_DB板块分(0-5) + SUB_THEME细分题材分
- 板块涨跌幅加分：>3%加1分，>1.5%加0.5分，<-2%扣0.5分
- 涨停验证：板块≥3只涨停加0.5分，1-2只加0.3分
- 资金流入：放量上涨(成交>500亿+涨>1%)加0.5分
- 生命周期：启动期/加速期/高潮期/震荡期/退潮期

### 2. 基本面 (0-5分, 权重30%)
- 基础评分：FwdPE+增速组合（低估值高增长=5分）
- PEG指标：<0.5加0.5分，>3扣0.5分，>4扣1分
- ROE：≥30%加0.5分，<3%扣0.5分
- 毛利率：≥50%高毛利护城河，<15%低毛利警示
- 估值阶梯：3年PE收缩趋势加分
- 行业前景：4档加分(+1/+0.5/0/-0.5)

### 3. 消息面 (0-5分, 权重20%)
- KNOWN_CATALYSTS已知催化剂（含来源/时间/情感标签/链接）
- iFinD新闻关键词分析（利好/利空词频统计）
- 利空覆盖：negative字段直接覆盖利好评分
- 股价确认：涨停/跌停作为消息验证

### 4. 技术面 (0-5分, 权重20%)
- 双线多头排列：日线MA5>MA10>MA20 且 周线WMA5>WMA10>WMA20
- 四级评分：双线多头(最高) > 仅周线 > 仅日线 > 均无
- BOLL通道位置：上轨/中轨/下轨 + 通俗解释
- 价格分位：120日高低点位置
- 量价配合：放量上涨/缩量下跌等6种模式
- 支撑压力位：BOLL轨道+均线+120日高低点
- 综合判断：强势/偏强/震荡/偏弱/弱势

## 组合推荐逻辑

从S/A级标的中筛选：
1. FwdPE ≤ 60
2. 增速 ≥ 15%
3. 价格分位 ≤ 95%
4. 板块去重（同板块仅选1只）
5. 综合评分加分：四维均衡(最低分≥3加0.5，≥4加1) + 中间位(40-80%加0.5)

## 常见问题处理

- **API限流**：脚本已内置串行请求+间隔控制，无需额外处理
- **数据缺失**：检查iFinD输出是否正常，KNOWN字典作为兜底
- **K线获取失败**：自动三重切换（腾讯→东方财富→新浪）
- **板块映射失败**：SECTOR_SINA_MAP维护自定义板块到新浪行业名称的映射
